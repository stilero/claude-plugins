# Default Issue Template

This is the fallback template used when no issue template is found in the target repository's `.github/` directory.

---

## Template

```markdown
## Summary

<!-- One or two sentences describing what this is and why it matters. -->

## Problem / Motivation

<!-- What is broken, missing, or costly? Be concrete. What happens today vs. what should happen? -->

## Proposed Solution

<!-- What would a good fix or implementation look like? Can be high-level. -->

## Value & Impact

<!-- Why is this worth doing? Who is affected? What gets better? -->

## Acceptance Criteria

- [ ] <!-- Specific, testable condition 1 -->
- [ ] <!-- Specific, testable condition 2 -->

## Context

<!-- Any relevant links, references, code snippets, or conversation context. -->

---
*Issue generated from conversation context.*
```

---

## Usage Notes

- Replace all `<!-- ... -->` comment placeholders with real content from the conversation
- Every section must be filled — never leave placeholder text in the final issue body
- Acceptance criteria must be specific and testable, not vague goals
- The "Context" section should include any relevant code snippets, file paths, or links mentioned in the conversation
